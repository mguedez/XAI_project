# XAI Project 

This project implements a **modular architecture for training, evaluating, and explaining Machine Learning models**, integrating classical *Explainable AI* (XAI) techniques such as **SHAP** and **LIME** together with an **LLM-based interpretative layer**.

The main objective is to study and compare **local and global explanations** across different datasets, while facilitating **reproducibility and extensibility** of the machine learning pipeline.

Currently, the project works with the **BCF** and **TIRESIA** datasets, using **Random Forest–based models** and automatically generated explanations.

---

# Project Objectives

## General Objective

To explore the use of **Large Language Models (LLMs) as an interpretative layer on top of Explainable AI (XAI) methods**, integrating them into Machine Learning pipelines to generate **natural language explanations** from technical outputs such as SHAP and LIME.

---

## Specific Objectives

* Implement **reproducible ML + XAI pipelines** for tabular datasets (BCF and TIRESIA).
* Train models and generate **local and global explanations** using SHAP and LIME.
* Integrate an **LLM** capable of interpreting XAI outputs and producing **natural language explanations**.
* Perform a qualitative analysis of the **fidelity and factuality** of the explanations generated by the LLM.

---

# Main Modules Description

## `src/pipelines/`

This module orchestrates the complete workflow.

* `bcf_pipeline.py`: end-to-end pipeline for the BCF dataset.
* `tiresia_pipeline.py`: end-to-end pipeline for the TIRESIA dataset.

Each pipeline integrates:

1. Data loading
2. Data preprocessing
3. Model training
4. Explanation generation
5. LLM-based interpretation

---

# Experimentation Notebooks

* `BCF_experiments.ipynb`: interactive execution of experiments on the BCF dataset.
* `TIRESIA_experiments.ipynb`: analysis and evaluation of the pipeline applied to TIRESIA.

These notebooks serve as **experimental support and validation** for the pipelines implemented in `src/`.

# Requirements and Installation

Install dependencies with:

```bash
pip install -r requirements.txt
```

Configure environment variables in a `.env` file (for example, API keys for LLM services).

## Gemini API Key Setup

1. Copy `.env.example` to `.env` in the project root.
2. Define the API key:

```env
GEMINI_API_KEY=your_api_key_here
```

`src/llm/gemini_explainer.py` automatically loads this variable to initialize the Gemini client.
